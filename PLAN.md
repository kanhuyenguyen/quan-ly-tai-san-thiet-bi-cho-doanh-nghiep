# KẾ HOẠCH THỰC HIỆN ĐỀ TÀI

---
## Tên đề tài
Xây dựng hệ thống quản lý tài sản & thiết bị cho doanh nghiệp

---
## Mục tiêu
Xây dựng hệ thống quản lý tài sản và thiết bị nhằm hỗ trợ việc theo dõi và cập nhật dữ liệu trong doanh nghiệp

---
## Mô tả hệ thống
Hệ thống cho phép người dùng:
- admin
  + Quản lý thông tin tài sản và thiết bị
  + Thêm, sửa và xóa dữ liệu
  + Theo dõi tình trạng tài sản và thiết bị
  + Tìm kiếm thông tin 
  + Hỗ trợ quản lý dữ liệu 
- user
  + Xem thông tin tài sản và thiết bị
  + Tìm kiếm thông tin 
  + Gửi yêu cầu / báo cáo tình trạng thiết bị
  
---
## Yêu cầu hệ thống
### Functional Requirements
- Đăng nhập / phân quyền người dùng (Admin / User)
- Thêm / Sửa / Xóa tài sản
- Thêm / Sửa / Xóa thiết bị
- Tìm kiếm tài sản và thiết bị
- Theo dõi tình trạng tài sản và thiết bị
- Hiển thị danh sách dữ liệu
### Non-functional Requirements
- Giao diện đơn giản, dễ sử dụng
- Hệ thống hoạt động ổn định
- Dữ liệu được lưu trữ 
  
---
## Công nghệ sử dụng
- Frontend: HTML, CSS, JavaScript
- Backend: Python Flask
- Database: SQL Server
- Công cụ: GitHub, Visual Studio Code
  
---
## Kế hoạch thực hiện
### Tuần 1
- Tìm hiểu yêu cầu đề tài
- Xác định chức năng hệ thống
- Tạo repository trên GitHub
- Thiết kế cơ sở dữ liệu
- Thiết kế giao diện cơ bản
### Tuần 2
- Xây dựng chức năng đăng nhập + phân quyền người dùng
- Xây dựng chức năng quản lý tài sản
- Xây dựng chức năng quản lý thiết bị
- Xây dựng chức năng tìm kiếm và thống kê
- Kết nối database với hệ thống
### Tuần 3
- Kiểm tra chức năng hệ thống
- Sửa lỗi và tối ưu giao diện
- Hoàn thiện các chức năng
### Tuần 4
- Hoàn thiện báo cáo thực tập
- Hoàn thiện demo hệ thống
